# Siam Tulip Web AR

Web AR สำหรับนิทรรศการดอกกระเจียวแดง ใช้ 8th Wall World Tracking เพื่อทำงานผ่าน Safari บน iPhone และ Chrome/เบราว์เซอร์ที่รองรับบน Android โดยไม่ต้องติดมาร์กเกอร์ พร้อมหน้าแอดมินที่ `/admin`

## เส้นทาง

- `/` เลือกโหมดตามค่าจากแอดมิน ค่าเริ่มต้นคือ World Tracking
- `/world` โหมดหลักสำหรับ iPhone และ Android
- `/floor` โหมด WebXR รุ่นทดลองเดิมสำหรับ Android
- `/marker-ar` โหมด Hiro marker สำรอง
- `/admin` หน้าแก้ไขเว็บไซต์ ไม่มีปุ่มหรือลิงก์จากหน้าเว็บหลัก
- `/marker` หน้ามาร์กเกอร์สำหรับพิมพ์กรณีใช้โหมดสำรอง
- `/qr` หน้าสร้าง QR หลังทราบ URL ที่ Deploy

## การทำงานของ World Tracking

1. ผู้เข้าชมเปิดเว็บไซต์ผ่าน HTTPS และอนุญาตกล้อง/การเคลื่อนไหว
2. หันกล้องลงพื้นบริเวณกลางห้องและขยับโทรศัพท์ช้า ๆ
3. ระบบ SLAM สร้างตำแหน่งโลก โดยกำหนดพื้นเป็นระดับ `Y=0`
4. ระบบยิงรังสีจากกึ่งกลางกล้องไปยังพื้นและแสดงวงแหวนตำแหน่ง
5. เมื่อถือนิ่งตามเวลาที่กำหนด ระบบวางดอกกระเจียวอัตโนมัติ
6. ใช้ **ย้ายตำแหน่ง** เพื่อวางใหม่ หรือ **ตั้งพื้นใหม่** เพื่อรีเซ็ต World Origin

โหมดนี้ไม่ต้องใช้ภาพห้องเป็นมาร์กเกอร์ และไม่ต้องติดกระดาษหรือสติกเกอร์บนพื้น ภาพห้องใช้เป็นเพียงแนวอ้างอิงตำแหน่งหน้างาน

## สิ่งที่ปรับได้จากแอดมิน

- เลือก `8th Wall World Tracking` หรือ `Hiro marker`
- เปิดหรือปิดการวางอัตโนมัติ และปรับเวลาหน่วงก่อนวาง
- ปรับความสูงกล้องเริ่มต้นและระยะวางไกลสุด
- ปรับสเกล ขนาด ตำแหน่ง ความสูง และการหมุนของวิดีโอ AR
- เปิด fallback ไปโหมดมาร์กเกอร์สำหรับอุปกรณ์ที่ไม่รองรับ
- เปิดหรือปิดนิทรรศการ
- ชื่อ คำอธิบาย ปุ่ม และข้อความสถานะทั้งหมด
- สีหลัก สีพื้นหลัง สีการ์ด สีข้อความ ความโค้ง และความทึบ
- รูปหน้าปกและวิดีโอ AR จากไฟล์ในเครื่องหรือ URL
- สี Chroma Key, Similarity, Smoothness และ Spill
- Draft/Publish, Import/Export JSON และคืนค่าเริ่มต้น

## การทำงานก่อนเชื่อม Supabase

- การตั้งค่า Draft และ Published เก็บใน `localStorage`
- รูปและวิดีโอที่อัปโหลดจากแอดมินเก็บใน `IndexedDB`
- ค่าจะมีผลเฉพาะ origin และอุปกรณ์/เบราว์เซอร์ที่แก้ไข
- ปุ่ม **เปิดพรีวิวเต็มหน้า** เปิด `/?preview=draft`
- ปุ่ม **เผยแพร่ในเครื่องนี้** ทำให้ `/` อ่านค่าชุด Published

เมื่อเชื่อม Supabase ภายหลัง ให้แทน storage layer ใน `js/settings-store.js` และ `js/media-store.js` โดยไม่ต้องรื้อหน้า UI

## วิธีทดสอบ

1. Auto Deploy ถูกปิดอยู่ ให้เข้า Vercel แล้ว Redeploy จาก `main` ด้วยตนเอง
2. เปิด URL Production ผ่าน HTTPS
3. iPhone ใช้ Safari และ Android ใช้ Chrome เป็นหลัก
4. อนุญาตกล้อง รวมถึง Motion/Orientation เมื่อระบบถาม
5. หันกล้องลงพื้นกลางห้องและขยับโทรศัพท์ช้า ๆ
6. เมื่อวงแหวนจับตำแหน่งนิ่ง ระบบจะวางดอกกระเจียวอัตโนมัติ

## หมายเหตุ

- ระบบต้องโหลด 8th Wall Engine Binary, XRExtras และ Landing Page จาก CDN
- ต้องเปิดผ่าน HTTPS เพื่อใช้กล้อง
- หลีกเลี่ยง Private Browsing และโหมดประหยัดพลังงานระหว่างทดสอบ
- วิดีโอพื้นหลังสีน้ำเงินถูกตัดด้วย GPU shader แบบเรียลไทม์
- `vercel.json` ยังคงตั้ง `git.deploymentEnabled` เป็น `false`
