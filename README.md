# Siam Tulip Web AR

Web AR สำหรับนิทรรศการดอกกระเจียวแดง พร้อมโหมดตรวจจับพื้นแบบไม่ต้องติดมาร์กเกอร์ และหน้าแอดมินที่ `/admin`

## เส้นทาง

- `/` เลือกโหมดตามค่าจากแอดมิน ค่าเริ่มต้นคือโหมดตรวจจับพื้น
- `/floor` ตรวจจับพื้นด้วย WebXR Hit Test แล้ววางดอกกระเจียว
- `/marker-ar` โหมด Hiro marker สำรอง
- `/admin` หน้าแก้ไขเว็บไซต์ ไม่มีปุ่มหรือลิงก์จากหน้าเว็บหลัก
- `/marker` หน้ามาร์กเกอร์สำหรับพิมพ์กรณีใช้โหมดสำรอง
- `/qr` หน้าสร้าง QR หลังทราบ URL ที่ Deploy

## การทำงานของโหมดตรวจจับพื้น

1. ผู้เข้าชมกดเริ่มและอนุญาต AR
2. หันกล้องลงพื้นบริเวณกลางห้องและขยับโทรศัพท์ช้า ๆ
3. ระบบยิง Hit Test จากกึ่งกลางหน้าจอเพื่อหาพื้น
4. เมื่อจับพื้นได้ต่อเนื่อง ระบบวางดอกกระเจียวอัตโนมัติ
5. ปุ่ม **ย้ายตำแหน่ง** ใช้กลับไปสแกนและวางใหม่

โหมดนี้ไม่ต้องใช้ภาพห้องเป็นมาร์กเกอร์ และไม่ต้องติดกระดาษหรือสติกเกอร์บนพื้น ภาพห้องใช้เป็นเพียงแนวอ้างอิงตำแหน่งหน้างาน

## เข้าแอดมินครั้งแรก

PIN เริ่มต้น: `2468`

ควรเปลี่ยน PIN ในหัวข้อ **ระบบและการสำรองค่า** ทันที

> PIN เวอร์ชันนี้เป็นเพียงการล็อกหน้า UI ฝั่งเบราว์เซอร์ ไม่ใช่ระบบรักษาความปลอดภัยจริง เมื่อเชื่อม Supabase ให้เปลี่ยนเป็น Supabase Auth และ RLS

## สิ่งที่ปรับได้จากแอดมิน

- เลือก `ตรวจจับพื้น` หรือ `Hiro marker`
- เปิดหรือปิดการวางอัตโนมัติ และปรับเวลาหน่วงก่อนวาง
- ปรับสเกล ขนาด ตำแหน่ง ความสูง และการหมุนของวิดีโอ AR
- เปิด fallback ไปโหมดมาร์กเกอร์สำหรับอุปกรณ์ที่ไม่รองรับ
- เปิดหรือปิดนิทรรศการ
- ชื่อ คำอธิบาย ปุ่ม และข้อความสถานะทั้งหมด
- สีหลัก สีพื้นหลัง สีการ์ด สีข้อความ ความโค้ง และความทึบ
- รูปหน้าปกและวิดีโอ AR จากไฟล์ในเครื่องหรือ URL
- สี Chroma Key, Similarity, Smoothness และ Spill
- Draft/Publish, Import/Export JSON และคืนค่าเริ่มต้น

## การทำงานก่อนเชื่อม Supabase

- การตั้งค่า Draft และ Published เก็บใน `localStorage`
- รูปและวิดีโอที่อัปโหลดจากแอดมินเก็บใน `IndexedDB`
- ค่าจะมีผลเฉพาะ origin และอุปกรณ์/เบราว์เซอร์ที่แก้ไข
- ปุ่ม **เปิดพรีวิวเต็มหน้า** เปิด `/?preview=draft`
- ปุ่ม **เผยแพร่ในเครื่องนี้** ทำให้ `/` อ่านค่าชุด Published

เมื่อเชื่อม Supabase ภายหลัง ให้แทน storage layer ใน `js/settings-store.js` และ `js/media-store.js` โดยไม่ต้องรื้อหน้า UI

## วิธีทดสอบโหมดพื้น

1. Deploy ขึ้น Vercel และเปิดผ่าน HTTPS
2. ใช้ Chrome บนมือถือ Android ที่รองรับ ARCore/WebXR
3. เปิด `/` แล้วกด **เริ่มเปิดกล้อง AR**
4. หันกล้องลงพื้นกลางห้องและขยับโทรศัพท์ซ้าย–ขวาช้า ๆ
5. เมื่อวงแหวนจับพื้นได้ ให้นิ่งไว้ประมาณ 0.9 วินาที
6. ดอกกระเจียวจะถูกวางตรงจุดกลางหน้าจอโดยอัตโนมัติ

## หมายเหตุ

- WebXR Hit Test ต้องเปิดผ่าน HTTPS และยังไม่รองรับเท่ากันทุกเบราว์เซอร์
- โหมดตรวจจับพื้นเหมาะกับ Android ที่รองรับ ARCore มากที่สุด
- หากอุปกรณ์ไม่รองรับ สามารถเปิด `fallbackToMarker` จากหน้าแอดมิน
- วิดีโอพื้นหลังสีน้ำเงินถูกตัดด้วย GPU shader แบบเรียลไทม์
- ระบบใช้ A-Frame 1.6.0 และเก็บไฟล์หลักไว้ในโฟลเดอร์ `vendor`
